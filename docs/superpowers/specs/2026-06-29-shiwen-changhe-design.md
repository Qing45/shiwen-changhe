## 1. 一句话定位

诗文长河是一款面向古诗词/历史爱好者的 Web 阅读应用，以"水墨长河"为核心视觉隐喻，把诗人与诗作按时间排成一条横向长卷，让用户在沉浸式漫游中遇见唐诗。

## 2. 背景与问题

市面现有古诗词应用（西窗烛、古诗文网、九歌、唐宋文学编年地图等）分两类：

- **工具型**（西窗烛、古诗文网）：按诗人列表或诗名搜索浏览，缺少时间维度的视觉化呈现
- **学术型**（唐宋文学编年地图）：信息密度高、阅读门槛高，不适合普通爱好者

用户痛点：

- 背诵时不知道诗人在历史中的位置、与其他诗人的时代关系
- 想看"同一时期的诗作密度"（盛唐诗人密集，晚唐稀疏）没有直观方式
- 现有应用把诗当成孤立文本，缺少"在时间长河中遇见诗"的体验

**机会**：把"时间线 + 长河视觉 + 沉浸阅读"结合，做一个为爱好者设计的产品。

## 3. 目标用户

**古诗词 / 历史爱好者**。非应试学生（课程过滤为未来扩展）。

## 4. MVP 范围

| 维度 | MVP 范围 | 未来扩展 |
|---|---|---|
| 内容 | 唐诗三百首（约 80 位诗人、300 首诗） | 全唐诗 → 宋词 → 元曲 |
| 朝代 | 仅唐（618–907） | 多朝代（唐→五代→北宋→南宋→元…） |
| 朝代细分 | 不做（无初/盛/中/晚唐） | 仅做正式历史朝代，不做文学分期 |
| 平台 | Web 网页优先 | 移动端响应式 |
| 图片 | 无题图 | 每首诗配水墨意境图 |
| 账号 | 无 | 收藏、笔记、背诵打卡 |
| 课程过滤 | 无 | 小学/初中/高中课本筛选 |

## 5. 三页面架构

```
主河（唐诗三百首全观）
  ├─ 搜索下拉 → 直接跳转
  └─ 点击诗人节点
        ↓
     支流页（该诗人全部诗作的长河）
        └─ 点击诗作节点
              ↓
           展开页（原文 + 注释 + 创作背景）
              └─ 上一首 / 下一首（诗人内翻页）
```

**关键决策**：搜索不是独立页面，而是主河顶部搜索框触发的下拉浮层。架构因此从四页面简化为三页面。

## 6. 视觉系统：月夜水墨

### 色彩

```
背景渐变   linear-gradient(180deg, #050818 0%, #0a1430 40%, #152548 70%, #0a1430 100%)
河流银线   linear-gradient(90deg, transparent, rgba(216,224,240,0.5), rgba(240,244,255,0.85), ...)
主文（月白） #e8f0ff
次文        #a8b8d8 / #8a98b8
辅助        #6478a0（远山墨）/ #5a6885（最弱）
高亮        #fff + box-shadow 多层发光
```

### 字体

- 中文：`KaiTi / STKaiti`（楷书）
- 数字：默认 sans-serif

### 视觉元素

- **河流主线**：横向银色发光线（垂直居中，2px 主线 + 10px blur 光晕）
- **节点**：圆形 pearl，按重要性放大（焦点诗人/诗作最大最亮）
- **远山墨影**：底部 radial-gradient ellipse，营造水墨意境
- **月亮**：右上角 radial-gradient + box-shadow 光晕
- **星点**：sparse radial-gradient circles

### 字号强约束（≥14px）

```
节点次要标签 / 元信息   14px
节点主标题             16–26px（按重要性）
原文                   18px / 行高 2.6 / 字距 2px
章节标题（注释/背景）   15–16px
```

## 7. 数据模型（最小）

```typescript
interface Dynasty {
  id: string;          // 'tang'
  name: string;        // '唐'
  startYear: number;   // 618
  endYear: number;     // 907
}

interface Poet {
  id: string;
  name: string;             // '李白'
  courtesyName?: string;    // 字 '太白'
  pseudonym?: string;       // 号 '青莲居士'
  birthYear: number;        // 701
  deathYear: number;        // 762
  dynastyId: string;        // 'tang'
  familiarity: number;      // 1–5，决定节点大小
}

interface Poem {
  id: string;
  title: string;            // '将进酒'
  poetId: string;
  content: string;          // 原文
  annotations: { term: string; explanation: string }[];
  background?: string;      // 创作背景
  creationYear?: number;    // 创作年份（可空，未知则按诗人中段排）
  familiarity: number;      // 1–5，决定节点大小
}
```

搜索索引覆盖：诗人名 / 诗名 / 诗句全文。

## 8. 页面规格

### 8.1 主河页面
Mockup: `navigation-v4.html`

- 顶部导航（高约 56px）：Logo "诗文长河" + 搜索框 + 朝代标签 "唐"
- 长河区域（剩余高度）：
  - 背景：远山墨影 + 月亮 + 星点
  - 河流主线：垂直居中
  - 诗人节点：按 `birthYear` 从左到右排
  - 节点尺寸：按 `familiarity` 微调（李白/杜甫最大）
  - 底部时间轴："618 · 唐 ─── 907"
- 横向滚动：拖动 / 滚轮 / 触控板

### 8.2 搜索下拉（主河内交互）
Mockup: `search-dropdown.html`

- 触发：搜索框获焦或输入字符
- 浮层位置：搜索框正下方，宽度对齐（约 440px）
- 内容：诗人 / 诗名 / 诗句 三段分组
- 每段：标题 + 计数 + 前 3 条 + "还有 N 条 ↓"
- 命中字符：白色 + `text-shadow` 发光
- 主河变暗（`filter: brightness(0.45)`）让浮层成为焦点
- 关闭：Esc / 点击外部 / 清空搜索框
- 点击结果：诗人 → 支流页；诗名 / 诗句 → 展开页

### 8.3 支流页（单个诗人）
Mockup: `poet-detail.html`

- 顶部导航：← 返回 + 诗人名（大）+ 字号 + 生卒 + 朝代标签
- 长河区域：延续主河视觉
  - 河流主线
  - 诗作节点：按 `creationYear` 排，**只显示诗名**（无金句、无年份、无人生阶段）
  - 焦点诗（如将进酒）：放大居中
  - 底部时间轴：诗人自己的生卒年（701—762）
- 横向滚动

### 8.4 展开页（单首诗）
Mockup: `poem-page.html`

- 顶部导航：← 返回诗人 + 诗名 + 作者·年份
- 内容（纵向滚动）：
  - 月夜氛围带（70px，月亮 + 星点，纯 CSS，**无题图**）
  - 原文（KaiTi 18px，居中，行高 2.6）
  - 注释（字段名白 + 解释淡）
  - 创作背景（纯文字段落）
  - 底部翻页卡：上一首 / 下一首
- 翻页范围：限当前诗人内，按 `creationYear`
- 边界处理：第一首时左卡置灰，最后一首时右卡置灰

## 9. 交互逻辑

| 触发 | 行为 |
|---|---|
| 主河点击诗人节点 | 跳转 `/poet/:poetId` |
| 主河点击搜索结果（诗人） | 跳转 `/poet/:poetId` |
| 主河点击搜索结果（诗名/诗句） | 跳转 `/poem/:poemId` |
| 支流页点击诗作节点 | 跳转 `/poem/:poemId` |
| 展开页底部翻页 | `/poem/:prevId` 或 `/poem/:nextId`（同诗人） |
| Esc / 点击浮层外 | 关闭搜索下拉 |
| 主河/支流横向滚动 | 拖动 + 滚轮 + 触控板 |

## 10. 未来扩展（不在 MVP）

- **多朝代扩展**：扩展到宋元明清后，主河自动在朝代更替处显示分界标签（如"唐 → 五代 → 北宋"）
- **图片题图**：每首诗配水墨意境图（替换展开页顶部的纯 CSS 氛围带）
- **课程过滤**：按小学/初中/高中课本筛选
- **用户账号**：收藏、笔记、背诵打卡
- **朝代快跳**：仅当多朝代时顶部出现朝代标签栏

## 11. 明确不做（YAGNI）

- 皇帝/重要历史事件轨道（用户已确认放弃）
- 朝代内文学分期（初/盛/中/晚唐，用户已确认不做）
- 缩放式 Semantic Zoom（总览+局部双视图）
- 用户评论 / UGC
- 音频朗诵
- 移动端原生 app（仅 Web 响应式）

## 12. 关键设计决策记录

| 决策 | 选择 | 原因 |
|---|---|---|
| 长卷方向 | 横向 | 与中国传统长卷一致，适合时间序列 |
| 视觉风格 | 月夜水墨（D 方案） | 兼具水墨意境与星河沉浸感 |
| 顶部导航 | Logo + 搜索框 + 唐 | 极简，无朝代快跳（MVP 单朝代） |
| 朝代处理 | 仅正式历史朝代，不做文学分期 | 初/盛/中/晚唐是文学批评概念，非正式朝代 |
| 搜索形态 | 主河内嵌下拉浮层 | 不跳离主河，三页面架构更简洁 |
| 支流页节点 | 只显示诗名 | 用户反馈：去掉人生阶段、金句、年份等附加信息 |
| 展开页题图 | MVP 不放 | 用户反馈：先不要图 |
| 展开页翻页 | 限当前诗人内 | 阅读连贯性：在同一诗人的语境中翻 |
