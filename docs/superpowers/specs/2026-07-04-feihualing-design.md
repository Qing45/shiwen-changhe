# 飞花令玩法 设计文档

> **状态**：已确认，待写实施计划
> **日期**：2026-07-04
> **作者**：用户与 Claude 协同设计
> **范围**：在现有「诗文长河」SPA 中加入飞花令玩法，含单人闯关 + AI 对战两个模式

---

## 1. 背景与目标

### 1.1 玩家目标
让用户在浏览唐诗之外，能以**游戏化方式**主动回忆诗句、加深印象，把"被动阅读"扩展为"主动运用"。

### 1.2 产品目标
复用现有 320 首唐诗题库与卷轴/夜空视觉语言，新增一个**纯前端、无后端依赖**的飞花令模块。实现成本低、可独立游玩、答错后能跳转原文阅读页形成学习闭环。

### 1.3 调研结论（市面飞花令 App 简述）
- **头部产品**：西窗烛（百万库 + 飞花令/接龙/答题三件套 + AI 助手）、飞花令·古诗词答题、飞花诗词
- **主流模式**：单人闯关（填空 / 选择 / 九宫格 / 看图）、AI 对战、本地双人轮流
- **常见痛点**：题库太小、AI 难度两极、缺学习性反馈、繁简体判定死板
- **本项目差异化**：题库 320 首（不大但精）、纯前端、卷轴视觉、答错一键跳转 PoemPage 看完整出处

### 1.4 范围与非目标
**范围内**：
- 单人闯关：50 关关键字递进解锁，九宫格 + 填空
- AI 对战：3 档难度，选择题，单局淘汰
- 共用关键字索引与判定引擎
- localStorage 进度持久化
- 完整视觉与现有页面协调

**非目标**：
- 多人对战 / 联网功能（无后端）
- 关键字位置限制的「超级飞花令」
- 反义词飞花令、诗词接龙（留作后续扩展）
- 玩家任意打字输入（题库外判定不可行）

## 2. 信息架构

### 2.1 新增路由
```
/play                 ← 飞花令大厅：模式选择 + 关卡地图
/play/stage/:kw       ← 单人闯关局（关键字 kw，如 "春"）
/play/ai/:kw          ← AI 对战局（关键字 kw）
```

### 2.2 入口
顶部 TopNav 第三 tab「飞花令」，与"诗人 / 诗文"并列。RiverToggle 由两档变三档：`诗人 · 诗文 · 飞花令`。飞花令页面继承夜空主题。

### 2.3 大厅布局
上下两个 tab：

**「闯关」tab**
- 顶部进度条：`12 / 50 关 · 已通 12 字`
- 主体：50 个关键字按难度排成 6×8 印章网格
  - 朱红印章 = 已通关
  - 灰白印章 = 未解锁（不可点）
  - 金边脉冲（CSS `focal-pulse` 动画） = 当前关
- 点击当前关 → `/play/stage/:kw`

**「对战」tab**
- 第一栏：选关键字（已通关的字 + 自由 5 字礼包：春/月/花/风/雪）
- 第二栏：选 AI 难度（青灯 / 墨客 / 诗圣）三选一
- 「开战」按钮 → `/play/ai/:kw?difficulty=...`

### 2.4 关卡解锁规则
- **通关条件**：在该关键字下答出 **5 句不重复**的诗句
- **解锁规则**：通关后解锁清单中下一个关键字（按预设难度顺序）
- **持久化**：`localStorage.shiwen-feihua-progress`

## 3. 单人闯关循环

### 3.1 每局流程（关键字 kw，5 句诗）

1. **入场**：卷轴展开动画（0.6s）。中央 120px 楷书关键字。右上角进度 `2/5`，左上角 3 滴血图标（❤❤❤），右上角倒计时环 120 秒。
2. **出题**：从该字诗句库随机选 1 句（题库内未答过的）。
   - **挖字规则**：挖掉 2-3 字（必含关键字 + 1-2 个干扰字）。其余字正常显示。
   - **示例**：关键字「春」，原句「春眠不觉晓，处处闻啼鸟」→ 展示 `□ 眠 □ 觉 □，处处闻啼鸟`。
3. **答题**：底部九宫格（12 字块）= 挖掉的字 + 9 个干扰字（从同年代唐诗高频字抽取，避开正确字）。玩家依次点击填空。
4. **判定**：
   - **答对**：得 1 分，1 秒金粉飘落庆祝，进入下一题（重新计时 120 秒）
   - **答错**：扣 1 滴血，正确答案高亮 1.5 秒后清空重答（不跳过本题）
   - **超时（120 秒）**：扣 1 滴血，同答错处理
5. **血量归零**：失败结算页，列出已答对 / 错过的诗句，可点击跳转 PoemPage 看完整出处
6. **答完 5 句**：通关动画（印章盖落 + 卷轴收起 0.8s），解锁下一关，返回大厅

### 3.2 进度断点续传
- 失败时**保留**已答对的句子（如已答对 2/5 句，下次接着答第 3 句）
- 主动退出（Esc）同样保留进度
- 数据键：`shiwen-feihua-progress.cleared[kw] = { correct: ["句1","句2"], failed: [...] }`

### 3.3 退出
- Esc → 返回大厅
- 浏览器关闭 → 进度自动保存到 localStorage

## 4. AI 对战循环

### 4.1 入场
选定关键字 + AI 难度后进入。布局为双栏对坐：

```
┌─────────────────────────────────────┐
│       关键字大字 居中「春」           │
│       当前回合：你 / AI              │
├──────────────┬──────────────────────┤
│ 你的诗囊     │  AI 对手（剪影）     │
│ ───────────  │  ─────────────       │
│ □ 春眠不觉…  │  □ 春风又绿…         │
│ □ 红豆生春…  │  □ 春潮带雨…         │
│              │  难度：墨客          │
├──────────────┴──────────────────────┤
│  4 选 1 题板 + 30 秒倒计时           │
│  A 春江花月夜   B 春风不度玉门关      │
│  C 春城无处不飞花   D 春眠不觉晓      │
└─────────────────────────────────────┘
```

### 4.2 回合循环

1. **首回合随机**：抛硬币决定玩家先 / AI 先
2. **玩家回合**：底部 4 选 1 题板
   - 4 个选项**都是题库内含关键字的诗句**，混排已用 / 未用
   - 玩家任选一句**未用过的** → 答对，句子进入「你的诗囊」→ 轮到 AI
   - 玩家选了**已用过的** → 「重复」判负（飞花令核心规则）
   - 超时 30 秒 → 判负
   - 4 句全部已用过的极端情况 → 系统补足未用过的新句（保证至少 1 个可选）
3. **AI 回合**：
   - 思考延迟（按难度）+ 答出概率（按难度）
   - 答出 → 句子进入「AI 诗囊」→ 轮到玩家
   - 漏答 → **AI 判负**（玩家胜）

### 4.3 AI 难度参数

| 难度 | 思考延迟 | 答错率 | 视觉剪影 |
|------|---------|--------|----------|
| 青灯 | 3.0s | 30% | 书生背影 |
| 墨客 | 1.5s | 10% | 文士持卷 |
| 诗圣 | 0.8s | 0% | 李白侧脸 |

### 4.4 AI 选句策略
- 从该关键字题库里排除「已用集」（玩家答过 + AI 答过）
- 剩余里随机抽 1 句作为 AI 的应答
- 墨客 / 青灯档：按概率「故意漏掉」（判负，玩家有利）
- 诗圣档：保证抽到（除非题库已空，此时 AI 判负）

### 4.5 判负条件
- 玩家：超时 / 答错 / 答出已用句
- AI：题库空 / 按难度概率漏答
- 飞花令原版规则保留「重复淘汰」

### 4.6 结算页
- **胜负显示** + 用时
- **已答诗句对账**：你 vs AI 双栏，列出本局双方各自答出的诗句（带出处链接到 PoemPage）
- **「未答出的诗句」**：题库里**剩余未被任何一方用过的**诗句（让玩家学习拓宽），可点击跳转 PoemPage 看完整出处
- **按钮**：「再来一局」/「换关键字」/「返回大厅」
- **战绩持久化**：`localStorage.shiwen-feihua-record`

## 5. 共享核心引擎

### 5.1 关键字索引（启动时一次性构建）
```typescript
// 离线扫描 320 首，构建 Map<关键字, Verse[]>
type Verse = { poemId: string; line: string; poetName: string; poemTitle: string };
function buildKeywordIndex(): Map<string, Verse[]>
```
- **扫描规则**：每首诗按标点切句，每句若包含关键字则记入索引
- **性能**：320 首 O(n) 扫描，启动约 10ms，可接受
- **缓存**：模块级变量，首次调用计算后复用

### 5.2 判定函数
```typescript
function validateVerse(
  input: string,       // 玩家/AI 给出的诗句
  keyword: string,     // 当前关键字
  used: Set<string>,   // 已用句集合
  pool: Verse[]        // 该关键字题库
): { ok: true; verse: Verse } | { ok: false; reason: 'not-in-pool' | 'duplicate' }
```
- 题库内判定：input 必须严格等于 pool 中某句的 line
- 重复判定：input ∉ used

### 5.3 AI 选句
```typescript
function aiPickAnswer(
  pool: Verse[],
  used: Set<string>,
  difficulty: 'qingdeng' | 'mohe' | 'shisheng'
): { picked: boolean; verse?: Verse }
```
- 诗圣：必答（除非池空）
- 墨客：90% 概率答出，10% 漏答
- 青灯：70% 概率答出，30% 漏答

### 5.4 共用 UI 组件
- `<KeywordSeal>`：印章式关键字节点（朱红 / 灰白 / 金边三态）
- `<VerseCard>`：诗句展示卡（带出处链接）
- `<ResultPage>`：通用结算页（标题 + 已答列表 + 错过列表 + 按钮组）
- `<PaperScroll>`：卷轴外框（复用 PoemPage 的木轴 + 双金线 + 朱印）

## 8. 数据持久化

### 8.1 单人闯关进度
```typescript
// localStorage.shiwen-feihua-progress
{
  unlockedIndex: number,            // 当前解锁到第几关（0-49）
  cleared: string[],                // 已通关关键字清单
  current: {                        // 当前进行中的局（断点续传）
    keyword: string,
    correct: string[],              // 已答对的句子
    blood: number,                  // 剩余血量
  } | null,
}
```

### 8.2 AI 对战战绩
```typescript
// localStorage.shiwen-feihua-record
{
  qingdeng: { win: number, lose: number },
  mohe:    { win: number, lose: number },
  shisheng:{ win: number, lose: number },
}
```

### 8.3 降级策略
- localStorage 不可用（隐私模式 / 满）：降级为内存进度，本局结束丢失
- 解析失败：重置为初始状态，不抛错

## 9. 错误处理与边界

| 场景 | 处理 |
|------|------|
| 关键字题库 < 5 句 | 该关标记为「Bonus 关」，答出 N 句即可（N = 实际句数） |
| 关键字题库为 0 | 自动跳过，标记 cleared，解锁下一关 |
| 玩家答出非题库诗句 | 不可能（输入严格限制在题库内选择） |
| 4 选 1 题板未用过句不足 | 优先填题库内未用过句；若不足 4 个，允许混入已用过的句（视觉标灰，玩家选了判负）；最极端情况（题库几乎被用完）AI 直接判负 |
| 浏览器关闭中途 | 进度自动保存（每次答对 1 句即写 localStorage） |
| 移动端窄屏 | 关卡地图 6×8 → 4×13；卷轴内字号自适应缩放 |
| 玩家清空 localStorage | 进度重置为初始（春关解锁） |

## 11. 实现要点

### 11.1 双源同步
本项目维护 `src/` + `scripts/build-standalone.cjs` 双源（前者用 TS + ESM，后者把同样代码内联进单一 HTML 文件用于离线分发）。飞花令所有新代码必须同步到两边。

### 11.2 现有可复用资产
- `fontFamilies.chinese`（楷书字体栈）
- `colors.bgGradient`、`PAPER_BG`、`PAPER_TEXT` 等配色常量
- `extractVariants` / `splitIntoLines`（诗句切分，已有）
- `getPoems` / `getPoet` / `getPoem`（数据访问层）
- `useVisited` hook（localStorage 模式参考）
- CSS keyframes：`twinkle`、`focal-pulse`、`river-flow` 等

### 11.3 文件结构规划
```
src/
├─ pages/
│   ├─ PlayHall.tsx          ← 飞花令大厅（关卡地图 + 对战配置）
│   ├─ StagePlay.tsx         ← 单人闯关局
│   └─ AiPlay.tsx            ← AI 对战局
├─ play/
│   ├─ keywords.ts           ← 关键字清单 + 难度分组
│   ├─ engine.ts             ← buildKeywordIndex / validateVerse / aiPickAnswer
│   ├─ progress.ts           ← localStorage 进度读写
│   └─ types.ts              ← Verse / Difficulty / PlayState 等
├─ components/
│   ├─ KeywordSeal.tsx       ← 印章节点
│   ├─ VerseCard.tsx         ← 诗句卡片
│   ├─ AiSilhouette.tsx      ← AI 剪影 SVG
│   ├─ NineGrid.tsx          ← 九宫格输入
│   └─ ChoiceBoard.tsx       ← 4 选 1 输入
└─ App.tsx                   ← 路由表加 3 条新路由
```

### 11.4 实施分阶段
1. **P1 核心引擎**：keywords + engine + progress + 单元测试
2. **P2 单人闯关**：PlayHall 闯关 tab + StagePlay 完整循环
3. **P3 AI 对战**：PlayHall 对战 tab + AiPlay 完整循环
4. **P4 视觉打磨**：动画、剪影、印章、移动端适配
